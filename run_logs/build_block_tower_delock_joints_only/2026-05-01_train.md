## Evaluation

The two components are coupled: regularization preserves the visual grounding that CPG relies on. The paper's Table 2 ablations show regularization alone fixes concept generalization, but CPG without regularization fails everything. So eval in layers:

### 1. Regularization alone — concept generalization (testable independently)

Serve this checkpoint in standard mode (no CPG) and prompt with **novel concepts** not seen during post-training (e.g. different block colours, unseen objects). Compare against the baseline joints-only checkpoint served the same way. If this checkpoint handles novel concepts better, the drift penalty is preserving visual grounding. This matches "DeLock w/o CPG" in Table 2.

```
uv run scripts/serve_policy.py policy:checkpoint --policy.config=pi05_build_block_tower_delock_6mix_joints_only --policy.dir=<checkpoint_dir>
```

### 2. CPG — spatial generalization (only works on top of regularization)

CPG steers the denoising flow away from the trained spatial bias. It only works when the visual encoder still has intact grounding (i.e. on this checkpoint, not on the baseline). Test by calling `Policy.infer` with a positive (novel spatial target) and negative (trained spatial target) prompt on **this** checkpoint:

```python
result = policy.infer(
    obs,                          # observation with the novel spatial prompt (τ⁺)
    negative_obs=negative_obs,    # same images/state, but with the trained prompt (τ⁻)
    cpg_guidance_scale=w,         # w=1.0 recovers positive-only; w>1.0 steers harder
)
```

Compare DeLock+CPG vs DeLock-without-CPG on novel spatial targets. Running CPG on the baseline checkpoint is not meaningful — without preserved grounding the contrastive signal has nothing to latch onto ("DeLock w/o Vis-Reg" in Table 2 fails broadly).

### 3. Full DeLock

DeLock checkpoint + CPG on prompts requiring both novel concepts and novel spatial targets. This is the full method.

The CPG path lives in `Pi0.sample_actions_cpg`, routed by `Policy.infer` when both `negative_obs` and `cpg_guidance_scale` are provided.
